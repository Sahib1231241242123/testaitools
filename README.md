## Русский

### Что это

OrderFlow - мини-платформа для аналитики заказов:

- импорт заказов в RetailCRM
- синхронизация CRM -> Supabase
- дашборд KPI и графиков на Next.js
- Telegram-алерты для крупных заказов
- cron endpoint для автоматического запуска

### Возможности

- современный дашборд: метрики, динамика, статусы, таблица заказов
- скрипты для импорта, синхронизации и алертов
- upsert в Supabase с защитой от дублей
- антидубли алертов через таблицу `order_alerts`

### Технологии

- Frontend: Next.js 15, React 18, Chart.js
- Скрипты: TypeScript + tsx
- База: Supabase (PostgreSQL)
- CRM: RetailCRM API v5
- Уведомления: Telegram Bot API

### Быстрый старт

1. Установить зависимости:

```bash
npm install
```

2. Создать `.env` по шаблону `.env.example` и заполнить реальными ключами.

3. Запустить дашборд:

```bash
npm run dev
```

Открыть: `http://localhost:3000`

### Полный туториал

1. Создайте таблицы в Supabase:

```sql
create table if not exists orders (
  id bigint generated by default as identity primary key,
  retailcrm_id bigint unique not null,
  external_id text,
  order_number text,
  created_at text,
  customer_name text not null,
  phone text,
  email text,
  city text,
  address text,
  status text,
  source text,
  total_kzt numeric default 0,
  raw_payload jsonb,
  synced_at timestamptz default now()
);

create table if not exists order_alerts (
  id bigint generated by default as identity primary key,
  retailcrm_id bigint unique not null,
  total_kzt numeric not null,
  created_at timestamptz default now()
);
```

2. Заполните `.env`:

- `RETAILCRM_BASE_URL`, `RETAILCRM_API_KEY`, `RETAILCRM_SITE`
- `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`
- `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`
- `HIGH_VALUE_THRESHOLD_KZT`, `CRON_SECRET`

3. Импортируйте мок-заказы в CRM:

```bash
npm run import:mock
```

4. Синхронизируйте CRM в Supabase:

```bash
npm run sync:orders
```

5. Запустите алерты для крупных заказов:

```bash
npm run watch:high-value
```

6. Проверьте дашборд в браузере.

### Скрипты

| Команда | Назначение |
|---|---|
| `npm run dev` | Запуск Next.js в dev |
| `npm run build` | Production build |
| `npm run start` | Запуск production build |
| `npm run import:mock` | Импорт заказов в RetailCRM |
| `npm run sync:orders` | Синхронизация RetailCRM -> Supabase |
| `npm run watch:high-value` | Telegram-алерты по крупным заказам |

### Cron endpoint

- `GET /api/cron/high-value`
- Header: `Authorization: Bearer <CRON_SECRET>`

### Troubleshooting

- Ошибка резолва `@/lib/supabase`: проверьте alias в `tsconfig.json`.
- Ошибки SSL/TLS: проверьте корректность `RETAILCRM_BASE_URL`.
- Нули в дашборде: выполните `npm run sync:orders`.

### Безопасность

- не коммитьте `.env`
- держите `SUPABASE_SERVICE_ROLE_KEY` только на серверной стороне
- при утечке ключей сразу ротируйте их

## English

### What is this

OrderFlow is an end-to-end analytics mini platform:

- import mock orders into RetailCRM
- sync CRM data to Supabase
- visualize KPIs and charts in Next.js
- send Telegram alerts for high-value orders
- trigger checks via cron endpoint

### Features

- modern dashboard with KPI cards, trends, status charts, and recent orders table
- scripts for import, sync, and alerting
- Supabase upsert flow with duplicate-safe behavior
- anti-duplicate alerts via `order_alerts`

### Tech Stack

- Frontend: Next.js 15, React 18, Chart.js
- Scripts: TypeScript + tsx
- Database: Supabase (PostgreSQL)
- CRM: RetailCRM API v5
- Notifications: Telegram Bot API
